# 插件市场安装功能测试指南

## 功能说明

现在插件市场支持两种安装方式：

### 方式 1：列表直接安装 ✅
- 后端列表接口已返回 `downloadUrl` 等字段
- 每个插件卡片右下角会显示蓝色的"安装"按钮
- 点击即可直接安装，无需查看详情

### 方式 2：详情页安装 ✅
- 点击插件卡片查看详情
- 详情弹窗右下角有"安装插件"按钮
- 显示完整的插件信息（版本、大小、更新说明等）

## 测试步骤

### 1. 启动应用
```bash
npm run dev
```

### 2. 打开插件市场
- 进入应用的插件管理页面
- 切换到"插件市场"标签

### 3. 检查列表中的安装按钮

**预期效果**：
- 每个插件卡片右下角应该显示：
  - 分类标签（如"productivity"）
  - 蓝色的"安装"按钮（带下载图标）

**如果看不到安装按钮**，可能的原因：
1. 后端返回的数据中 `downloadUrl` 为空或 `null`
2. 网络请求失败
3. 浏览器缓存问题

### 4. 测试列表安装

**操作**：
1. 点击任意插件卡片上的"安装"按钮
2. 按钮文字变为"安装中..."
3. 等待下载和安装完成
4. 按钮变为"已安装"（带勾选图标）
5. 收到系统通知"安装成功"

**注意**：
- 安装按钮会阻止事件冒泡，不会触发详情弹窗
- 安装过程中按钮会被禁用

### 5. 测试详情页安装

**操作**：
1. 点击插件卡片的其他区域（不是安装按钮）
2. 弹出详情窗口
3. 查看完整信息：
   - 插件图标和名称
   - 详细描述
   - Stars、下载次数、文件大小
   - 更新说明
   - 标签
4. 点击右下角"安装插件"按钮
5. 等待安装完成
6. 详情窗口自动关闭
7. 插件列表刷新

## 调试方法

### 检查后端返回的数据

打开浏览器开发者工具（F12），查看 Network 标签：

1. 找到 `/api/v1/plugins` 请求
2. 查看 Response，确认每个插件对象包含：
```json
{
  "id": "translate",
  "name": "Translate",
  "downloadUrl": "https://github.com/.../plugin.zip",  // ← 必须有值
  "version": "v1.0.0",
  "size": 1048576,
  // ... 其他字段
}
```

### 检查前端状态

在浏览器控制台（Console）中：

```javascript
// 查看插件数据
console.log(plugins);

// 检查第一个插件是否有 downloadUrl
console.log(plugins[0]?.downloadUrl);
```

### 常见问题

**问题 1：看不到安装按钮**
- 原因：`downloadUrl` 为空或未定义
- 解决：检查后端是否正确返回该字段

**问题 2：点击安装没反应**
- 原因：可能是网络问题或下载链接无效
- 解决：打开控制台查看错误信息

**问题 3：安装失败**
- 原因：ZIP 文件格式错误或 manifest.json 缺失
- 解决：检查插件 ZIP 文件结构

**问题 4：提示"插件已存在"**
- 原因：插件已经安装过
- 解决：先卸载旧版本再安装

## 后端数据要求

确保后端列表接口返回以下字段：

```typescript
interface MarketplacePlugin {
  id: string;
  name: string;
  description: string;
  author: string;
  repository: string;
  icon: string;
  category: string;
  keywords: string[];
  addedAt: number;
  downloads: number;
  stars: number;
  
  // 安装必需字段
  downloadUrl?: string;  // 下载链接（必须）
  version?: string;      // 版本号
  size?: number;         // 文件大小（字节）
  checksum?: string;     // 校验和（可选）
  releaseDate?: string;  // 发布日期（可选）
}
```
