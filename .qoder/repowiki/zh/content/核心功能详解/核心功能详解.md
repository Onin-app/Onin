## 简介

Baize 是一个基于 Tauri + SvelteKit + TypeScript 构建的快速启动应用程序，旨在提供类似于 Raycast、UTools、Alfred 和 Wox 的功能体验。该项目采用现代化的技术栈，结合了 Rust 的高性能后端和现代 Web 技术的前端界面，为用户提供快速、响应迅速的应用程序启动和命令执行体验。

### 设计目标

- **快速启动**：通过智能搜索和索引机制，实现毫秒级的应用程序和命令启动
- **高度可扩展**：通过插件系统支持第三方功能扩展
- **跨平台兼容**：支持 Windows、macOS 和 Linux 平台
- **用户友好**：简洁直观的界面设计，符合用户习惯的操作方式

### 技术架构

项目采用分层架构设计：
- **前端层**：基于 SvelteKit 的响应式用户界面
- **后端层**：Rust 编写的高性能服务端逻辑
- **通信层**：通过 Tauri 实现前后端双向通信
- **插件层**：独立的 JavaScript/TypeScript 插件系统

## 窗口管理系统

窗口管理系统是 Baize 的核心组件之一，负责控制应用程序主窗口的生命周期、焦点管理和智能隐藏行为。

### 核心功能特性

1. **智能隐藏机制**：当窗口失去焦点时自动隐藏，提升用户体验
2. **文件拖拽支持**：在文件拖拽过程中保持窗口打开状态
3. **ESC 键处理**：通过全局快捷键和本地按键处理实现灵活的关闭机制
4. **焦点状态跟踪**：精确监控窗口焦点变化，避免不必要的隐藏操作

### 插件存储管理功能

最新版本中引入了插件存储管理功能，为插件提供了持久化数据存储能力。

#### 存储 API 接口

插件可以通过 `storage` 对象访问以下存储方法：

- **setItem(key, value)**: 设置单个键值对
- **getItem(key)**: 获取指定键的值
- **removeItem(key)**: 删除指定键
- **clear()**: 清空当前插件所有存储数据
- **keys()**: 获取当前插件所有键的数组
- **setItems(items)**: 批量设置多个键值对
- **getItems(keys)**: 批量获取多个键的值

```typescript
import { storage } from '@baize/plugins-sdk';

// 使用示例
async function example() {
  // 存储用户偏好设置
  await storage.setItem('theme', 'dark');
  
  // 存储复杂对象
  await storage.setItem('userSettings', {
    fontSize: 14,
    language: 'zh-CN'
  });
  
  // 批量存储
  await storage.setItems({
    lastVisit: Date.now(),
    visitCount: 1
  });
  
  // 读取数据
  const theme = await storage.getItem('theme');
  const settings = await storage.getItem('userSettings');
  
  // 批量读取
  const data = await storage.getItems(['lastVisit', 'visitCount']);
}
```

#### 存储数据隔离

每个插件的数据存储完全隔离，确保数据安全：

- **独立存储路径**：每个插件有独立的存储文件 `plugin_data/{plugin_id}/storage.json`
- **上下文识别**：通过线程本地存储（thread_local!）跟踪当前执行的插件ID
- **访问控制**：插件只能访问自己的存储空间，无法访问其他插件或主应用数据

```rust
// 获取插件存储路径
fn get_plugin_store_path(plugin_id: &str) -> String {
    format!("plugin_data/{}/storage.json", plugin_id)
}

// 线程本地存储用于保存当前插件ID
thread_local! {
    static CURRENT_PLUGIN_ID: std::cell::RefCell<Option<String>> = const { std::cell::RefCell::new(None) };
}
```

## 性能考虑

Baize 在设计时充分考虑了性能优化，采用了多种策略确保快速响应和高效资源利用。

### 启动性能优化

1. **预渲染**：使用 SvelteKit 的静态站点生成（SSG）技术
2. **延迟加载**：按需加载非关键功能模块
3. **缓存策略**：智能缓存命令列表和图标资源
4. **异步初始化**：后台异步加载大量数据

### 运行时性能优化

1. **内存管理**：使用 Rust 的所有权系统确保内存安全
2. **并发处理**：Tokio 异步运行时处理 I/O 密集型操作
3. **事件去抖**：减少高频事件的处理频率
4. **资源池化**：复用昂贵的资源对象

### 搜索性能优化

1. **索引预计算**：提前构建命令关键词索引
2. **模糊匹配算法**：高效的字符串匹配算法
3. **结果缓存**：缓存热门查询的结果
4. **增量更新**：只更新变化的部分

## 故障排除指南

### 常见问题诊断

1. **窗口无法显示**
   - 检查焦点管理设置
   - 验证全局快捷键注册状态
   - 确认系统托盘可用性

2. **快捷键不响应**
   - 检查权限设置（macOS 辅助功能）
   - 验证快捷键格式正确性
   - 确认没有与其他应用冲突

3. **插件加载失败**
   - 检查插件目录权限
   - 验证插件 manifest.json 格式
   - 查看插件错误日志

4. **命令执行异常**
   - 检查命令路径有效性
   - 验证权限设置
   - 确认依赖项可用性
