## 引言
Baize插件系统采用基于能力（Capability-based）的安全模型，通过声明式权限控制机制确保插件在提供强大功能的同时不损害宿主应用的整体安全性。该模型要求插件在`manifest.json`中明确声明其所需权限，并在运行时由宿主应用进行验证和强制执行。本文档全面记录该权限控制系统的架构、实现机制和使用方法。

## 权限声明机制
插件必须在其`manifest.json`文件中通过`permissions`字段声明其所需权限。系统已将原有的扁平化权限结构重构为更灵活、更直观的命名空间结构。现在支持细粒度的权限控制，每个权限模块都有独立的配置选项。

```json
{
  "id": "com.example.plugin",
  "name": "Example Plugin",
  "version": "1.0.0",
  "entry": "index.html",
  "permissions": {
    "http": {
      "enable": true,
      "allowUrls": ["https://api.example.com", "https://*.api.example.com"],
      "timeout": 30000,
      "maxRetries": 3
    },
    "storage": {
      "enable": true,
      "local": true,
      "session": false,
      "maxSize": "10MB"
    },
    "notification": {
      "enable": true,
      "sound": true,
      "badge": false
    },
    "command": {
      "enable": true,
      "allowCommands": ["my-command-*"],
      "maxExecutionTime": 5000
    }
  }
}
```

当插件加载时，宿主应用会解析此声明并将其与插件的实际行为进行比对，确保所有操作都在授权范围内。

## 权限配置与能力模型
系统通过`capabilities/`目录下的JSON文件定义不同场景下的能力集。`plugin.json`文件定义了插件窗口的权限集合，包括核心功能、WebView控制和窗口操作权限。

```json
{
  "identifier": "plugin-capability",
  "description": "插件窗口的能力",
  "windows": ["plugin_*"],
  "permissions": [
    "core:default",
    "core:webview:default",
    "core:webview:allow-internal-toggle-devtools",
    "core:window:default",
    "core:window:allow-close",
    "core:window:allow-hide",
    "core:window:allow-show",
    "core:window:allow-minimize",
    "core:window:allow-maximize",
    "core:window:allow-unmaximize",
    "core:window:allow-set-focus",
    "core:window:allow-set-title"
  ]
}
```

这些能力与插件声明的权限相结合，形成完整的权限控制策略。

## 权限验证与运行时执行
系统在多个层面执行权限验证。当插件尝试发起网络请求时，`check_http_permission`函数会首先检查`permissions.http.enable`是否启用，然后验证请求URL是否在`permissions.http.allowUrls`声明的允许列表中。

## 支持的权限类型与粒度
当前系统支持多种权限类型，具有细粒度的控制能力：

### HTTP权限
- **启用开关**: `http.enable` 控制是否启用HTTP权限
- **URL白名单**: `http.allowUrls` 支持精确域名匹配（`https://api.example.com`）和通配符子域名匹配（`https://*.api.example.com`）
- **超时设置**: `http.timeout` 可选参数，单位毫秒
- **重试次数**: `http.maxRetries` 可选参数，控制最大重试次数
- **完整验证**: 支持协议、端口和主机名的完整验证

### 存储权限
- **启用开关**: `storage.enable` 控制是否启用存储权限
- **存储类型**: `storage.local` 和 `storage.session` 分别控制本地存储和会话存储的访问
- **大小限制**: `storage.maxSize` 可选参数，设置最大存储容量
- **数据隔离**: 每个插件拥有独立的存储空间（`plugin_data/{plugin_id}/storage.json`），确保插件间无法互相访问存储

### 通知权限
- **启用开关**: `notification.enable` 控制是否启用通知权限
- **声音通知**: `notification.sound` 控制是否允许播放声音
- **徽章通知**: `notification.badge` 控制是否允许显示应用徽章

### 命令权限
- **启用开关**: `command.enable` 控制是否启用命令权限
- **命令白名单**: `command.allowCommands` 支持通配符匹配（`my-command-*`）
- **执行时间**: `command.maxExecutionTime` 可选参数，限制命令最大执行时间

## 基于能力的安全模型分析
Baize的权限控制系统采用基于能力的安全模型，有效平衡了插件功能性和系统安全性：

### 安全优势
- **最小权限原则**：插件只能访问其明确声明的资源
- **纵深防御**：多层验证机制（声明验证、运行时检查）
- **数据隔离**：每个插件有独立的存储和执行环境
- **透明控制**：用户可审查插件权限声明
- **显式启用**：每个权限模块都需要显式启用（`enable: true`）

### 功能性保障
- **灵活的权限粒度**：支持通配符和精确匹配
- **异步执行模型**：不影响主应用性能
- **统一API接口**：开发者无需关心底层安全机制
- **可扩展架构**：易于添加新的权限类型

### 运行时架构
系统通过Deno JS运行时为headless插件提供执行环境，并通过Tauri的IPC机制与webview插件通信。权限检查在Rust后端执行，确保不可绕过。

## 插件开发者指南
为正确使用权限系统，插件开发者应遵循以下指南：

### 1. 权限声明
在`manifest.json`中准确声明所需权限：

```json
{
  "permissions": {
    "http": {
      "enable": true,
      "allowUrls": [
        "https://api.service.com",
        "https://*.cdn.service.com"
      ],
      "timeout": 30000,
      "maxRetries": 3
    },
    "storage": {
      "enable": true,
      "local": true,
      "session": false
    },
    "notification": {
      "enable": true,
      "sound": true,
      "badge": false
    },
    "command": {
      "enable": true,
      "allowCommands": ["my-plugin-*"]
    }
  }
}
```

### 2. 权限迁移
对于现有插件，需要将旧的扁平化结构迁移到新的命名空间结构：

```json
// 旧的结构
"permissions": {
  "network": ["https://api.example.com"]
}

// 新的结构
"permissions": {
  "http": {
    "enable": true,
    "allowUrls": ["https://api.example.com"]
  }
}
```

### 3. API 使用
使用SDK提供的API进行权限操作：

```typescript
// 发起HTTP请求
await invoke('plugin_request', {
  url: 'https://api.example.com/data',
  method: 'GET'
});

// 显示通知
await invoke('show_notification', {
  title: '通知标题',
  body: '通知内容'
});

// 存储数据
await invoke('plugin_storage_set', {
  key: 'user-preferences',
  value: { theme: 'dark' }
});
```

### 4. 错误处理
妥善处理权限相关的错误：

```typescript
try {
  const response = await invoke('plugin_request', options);
  console.log('请求成功:', response);
} catch (error) {
  if (error.name === 'PermissionDeniedError') {
    console.error('权限被拒绝:', error.message);
    // 提示用户检查manifest.json中的权限声明
  } else {
    console.error('请求失败:', error);
  }
}
```

## 结论
Baize插件系统的权限控制模型通过引入命名空间结构的权限系统，实现了更精细、更灵活的安全控制。这种基于能力的安全模型不仅增强了系统的安全性，还为插件开发者提供了清晰的权限管理机制。通过在`manifest.json`中声明权限并在运行时强制执行，系统有效地平衡了插件的功能需求与应用的整体安全要求。
