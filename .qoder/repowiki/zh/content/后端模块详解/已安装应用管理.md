## 简介

`installed_apps` 模块是本项目中的一个核心功能组件，负责跨平台发现和管理系统上已安装的应用程序。该模块为前端应用提供了一个统一的接口，使其能够检索、显示和启动用户设备上的应用程序。本技术文档将深入分析该模块的架构设计、跨平台实现策略以及其与系统其他组件的集成方式。

该模块的设计遵循了Rust的模块化和条件编译特性，为Windows、macOS和Linux三大主流操作系统提供了各自独立的实现。通过一个门面（Facade）模式的公共接口，它向`command_manager`等上层模块隐藏了平台差异，确保了代码的可维护性和可扩展性。

## 核心组件分析

### 公共接口与数据结构

`mod.rs` 文件是整个模块的入口点，它定义了跨平台的公共接口和数据结构。核心是 `AppInfo` 结构体，它标准化了从不同平台获取的应用信息。

`fetch_installed_apps()` 函数是模块的核心异步函数，它利用Rust的条件编译（`#[cfg(target_os = "...")]`）在运行时选择并调用相应平台的实现，最终返回一个 `Vec<AppInfo>`。`open_app()` 命令则负责启动指定路径的应用程序。

## 跨平台实现策略

### Windows 实现

Windows平台的实现最为复杂，因为它需要从多个来源发现应用程序。`windows.rs` 模块综合运用了注册表查询、快捷方式扫描和UWP包枚举三种策略。

```mermaid
flowchart TD
Start([开始获取应用]) --> Registry["从注册表 HKEY 查询"]
Registry --> Shortcuts["扫描快捷方式 .lnk 文件"]
Shortcuts --> AppsFolder["枚举 AppsFolder (UWP)"]
AppsFolder --> Merge["合并应用列表"]
subgraph "注册表查询"
Registry --> |路径| HKEY_LOCAL_MACHINE
Registry --> |路径| HKEY_CURRENT_USER
HKEY_LOCAL_MACHINE --> |过滤| Filter1["过滤系统组件"]
HKEY_CURRENT_USER --> |过滤| Filter1
Filter1 --> |提取| Extract1["提取 DisplayName, DisplayIcon, InstallLocation"]
Extract1 --> Normalize["规范化应用名"]
Normalize --> |生成| App1["AppInfo (origin: Hkey)"]
end
subgraph "快捷方式扫描"
Shortcuts --> |遍历| LnkFiles["C:\\ProgramData\\Microsoft\\Windows\\Start Menu"]
LnkFiles --> |解析| ParseLnk["解析 .lnk 文件"]
ParseLnk --> |获取| TargetPath["目标可执行文件路径"]
TargetPath --> |生成| App2["AppInfo (origin: Shortcut)"]
end
subgraph "UWP 应用枚举"
AppsFolder --> |COM API| EnumUWP["SHGetKnownFolderItem(FOLDERID_AppsFolder)"]
EnumUWP --> |获取| UWPApp["UWP 应用信息"]
UWPApp --> |生成| App3["AppInfo (origin: Uwp)"]
end
Merge --> |优先级合并| FinalApps["最终应用列表"]
FinalApps --> End([返回应用列表])
```
